# School Route Protocol

Use this to infer a school's or program family's admissions route: the path by which an applicant can credibly fit, compensate, and stand out.

## Route Inputs

Use at least three classes of evidence when possible:

- official identity: program overview, mission, curriculum, tracks, student work, faculty, centers, career outcomes
- admissions mechanics: requirements, prerequisites, tests, portfolio, interview, deadlines
- public signals: admitted profiles, current student work, alumni reflections, forum posts, agency cases
- applicant match: hard profile, soft evidence, story, constraints, timing

## Route Dimensions

For each school/program family, assess:

- selection ideology: research, practice, product, design, quant, leadership, public impact, entrepreneurship
- cohort archetypes: technical builder, researcher, designer, practitioner, career switcher, founder, policy person, artist-engineer
- hard-gate strictness: GPA, prerequisites, tests, portfolio, work experience
- soft-evidence tolerance: projects, patents, portfolio, startup, unusual background, creative work
- proof style: transcript-heavy, portfolio-heavy, recommendation-heavy, statement-heavy, case/interview-heavy
- international applicant pattern: if evidence exists
- route risks: hidden prerequisites, cohort mismatch, low yield for cross-major applicants, high cost/low fit

## Route Types

Use one or more:

- `academic-route`: transcript, research, recommendations, and prerequisites dominate.
- `builder-route`: shipped systems, technical projects, product evidence, and implementation depth matter.
- `studio-route`: portfolio, taste, design process, critique readiness, and creative identity matter.
- `professional-route`: internships, leadership, client/project outcomes, and career clarity matter.
- `interdisciplinary-route`: cross-domain coherence and unusual evidence can compensate for nonstandard background.
- `elite-filter-route`: prestige, GPA, prior institution, and scarce seats create high variance even with good fit.

## Output Rule

Every route card must include:

```text
This program appears to reward [qualities] because [evidence].
The applicant fits through [evidence].
The applicant is exposed on [risks].
Best route: [route type].
```

If evidence is thin, mark the route as `hypothesis-low-confidence`.
