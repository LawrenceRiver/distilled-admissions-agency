# Admissions Workflow Visual Redesign

## Goal

Replace the README's image-heavy feature showcase with one professional, bilingual workflow graphic that explains what the skill does from intake to decision.

## Chosen direction

Use a clean five-step horizontal process diagram:

1. Applicant Passport / 申请者画像
2. Evidence Archive / 官网存证
3. Program Audit / 项目审计
4. Route Cards / 录取路线卡
5. Fit Matrix / 冲主保矩阵

Use restrained colors, white cards, a light background, one dark output band, and short bilingual labels. Keep the existing illustrative assets in the repository for provenance, but remove them from the main README path so the first read is structured rather than decorative.

## Scope

- Add `assets/admissions-workflow.svg` as a crisp, repository-native visual.
- Replace the main README hero and feature-cut image blocks with the workflow visual and a concise output explanation.
- Apply the same visual hierarchy to `README.en.md` and `README.zh-CN.md`.
- Add an MIT license to both repositories and link it from the README files.

## Constraints

- Do not claim that public cases prove admission probability.
- Keep official school pages as the source of truth.
- Keep applicant data local and out of the reusable skill package.
- Existing illustrative assets remain separate from the MIT-licensed source code unless their provenance is independently confirmed.
